# Contributing

Thanks for wanting to help! This page explains how to contribute code, rule suggestions, and art.

How to contribute
- Report bugs: open an Issue with steps to reproduce, expected vs actual behavior, and environment.
- Suggest rules/design changes: open an Issue with the proposal and rationale; include playtest notes if available.
- Code contributions: fork, create a feature branch, run tests, and open a PR.

Development workflow
1. Fork the repo and branch from main (or the default branch).
2. Make changes in a branch named `feature/<short-description>`.
3. Add tests and update docs when behavior changes.
4. Open a pull request describing the change, motivation, and testing done.

Code style and testing
- Follow the repository’s formatting rules (add Prettier/ESLint configs here).
- Include unit tests for rule changes; include reproducible scenarios for gameplay changes.

Templates
- Use ISSUE_TEMPLATE and PULL_REQUEST_TEMPLATE (examples are included in this docs folder and `.github/`).
- Add screenshots or GIFs for UI changes.

Communication
- For design discussions, open an Issue marked with the label `discussion`.
- Major rule changes should be discussed in Issues before implementation.

If you prefer, I can draft PRs for documentation changes or add `CODE_OF_CONDUCT.md` and `CONTRIBUTING.md` at the repo root.
